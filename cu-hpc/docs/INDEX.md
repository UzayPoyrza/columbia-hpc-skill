# Columbia HPC — Documentation Index (local)

Local copy of Columbia's RCS HPC documentation (https://columbiauniversity.atlassian.net/wiki/spaces/rcs), lightly corrected where we
verified fixes on-cluster. **These pages are the source of truth for facts.**

## How to use this index (navigation judgment)

- **Facts come from these pages; decisions come from the skill's principles.** Need a spec, a module
  name, a path, a limit, or how to run a specific application? Find the topic below, open **that one
  page**, and read the relevant part — don't recall it from memory and don't load everything.
- **Match the cluster.** Insomnia, Ginsburg, Terremoto and the Free Tier differ — open the page for the
  cluster the user is on.
- **When a page is silent or you're unsure, verify cheaply** in an interactive `srun` session rather
  than assume.
- For *how to size a job* or *the submit workflow*, use `../SKILL.md` and `../reference/resources.md`,
  not these pages.

## By task

| I need to… | Open |
|---|---|
| get access / log in / Duo / find my account | the cluster's **Getting Started** page |
| submit a job / resource directives / walltime & job limits | the cluster's **Submitting Jobs** page |
| know home vs scratch paths, quotas | the cluster's **Storage** page (on Ginsburg, read **New Scratch Space** too) |
| see installed software / module names | the cluster's **Software** page |
| node specs, GPUs, partitions | the cluster's **Technical Information** page |
| worked job examples (MPI, GPU, containers, MATLAB, Python, R, Julia, Jupyter) | the cluster's **Job Examples** page |
| run a specific app (Schrödinger, MRtrix, parallel MATLAB) | the matching **Running …** page below |
| maintenance windows, SLA, how to get help | the **General / cross-cluster** pages |
